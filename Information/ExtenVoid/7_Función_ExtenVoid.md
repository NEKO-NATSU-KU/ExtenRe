## ⚙️ How does ExtenVoid work?

The operation of **ExtenVoid** is based on a simple but structured workflow:

1. **Cloning the active project**: Start from an existing repository that you want to modify or improve.
2. **Analysis of the current state**: Identify the technologies used (Java, Gradle, APIs, etc.), their version, and possible improvement points.
3. **Application of controlled changes**:
   - Updating dependencies to newer versions.
   - Migrating obsolete APIs to new alternatives.
   - Changes in project configuration (e.g., updating Gradle wrapper).
   - Modifications to the source code to improve performance, security, or compatibility.
4. **Documentation of changes**: Each modification is recorded so that other developers can understand the process and replicate it if necessary.
5. **Testing and validation**: Tests are run to ensure the project still works correctly after the changes.
6. **Publication of the modified version**: The result is shared as a fork or branch, allowing the community to use it or contribute.

In summary, ExtenVoid acts as a **technical assistant** to keep alive and updated projects that are already functioning.
