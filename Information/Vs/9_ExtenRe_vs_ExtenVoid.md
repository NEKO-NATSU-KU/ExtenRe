## ⚖️ Differences between ExtenRe and ExtenVoid

Although both projects share roots and similar goals (keeping open source code alive), their approaches are different and complementary.

| Feature | ExtenRe | ExtenVoid |
|---------|---------|-----------|
| **Target project state** | Abandoned, suspended, or inactive | Active, under maintenance |
| **Main objective** | Rebuild from the base to give new life | Modify and improve on existing |
| **Type of changes** | Deep structural (changing Java versions, Gradle, obsolete APIs) | Specific changes, controlled updates, new functionalities |
| **Starting point** | Cloned repository of an abandoned project | Cloned repository of an active project |
| **Risk** | High (may require rewriting parts) | Low to medium (original structure preserved) |
| **Concrete example** | Rebuilding Inotia00's ReVanced Extended updating to Java 21 and Gradle 9.3.1 | Modifying an active app to change its payment API or update a library |

### In summary:
- **ExtenRe** = Resurrect what was dead.
- **ExtenVoid** = Perfect what already lives.
