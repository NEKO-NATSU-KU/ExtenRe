## 🤔 Which one do you recommend or which is better?

There is no single answer, because it **depends on the state of the project you want to work with**. Here's a guide to help you decide:

### Choose **ExtenRe** if:
- The project you're interested in **is abandoned** (no recent commits, unanswered issues, the original author no longer maintains it).
- You need **deep updates** like changing Java version, Gradle, or migrating obsolete APIs.
- You want to **resurrect** a project that you believe still has value but needs a solid foundation to move forward.
- You're willing to **rebuild parts** of the code to adapt it to current standards.

### Choose **ExtenVoid** if:
- The project **is active or under maintenance** but requires **specific improvements** (updating a library, fixing a bug, adding a feature).
- You want to **contribute** to an existing project without altering its fundamental structure.
- You need to **adapt** a project to new technologies without breaking backward compatibility.
- You prefer **controlled and reversible changes** that you can document and share easily.

### And if you're not sure?
Ask yourself: **Does the project still have a pulse?**
- If yes (there's recent activity, community, or the author responds), start with **ExtenVoid**.
- If no (everything indicates it's dead), then **ExtenRe** is your best option.

Both projects are **complementary**: a project resurrected with ExtenRe can then be kept alive with ExtenVoid. They form a complete ecosystem for the longevity of open source code!
